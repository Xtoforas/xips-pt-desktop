# Phase 3: Today Page Redesign

## Goal

Turn the landing page into the primary operating surface for readiness, blockers, active work, and recent outcomes.

## Scope

- [ ] Replace the current metric-card-first overview with a prioritized landing page.
- [ ] Add a top readiness strip for server, auth, watch-folder coverage, and automation-rule status.
- [ ] Add a prominent `Needs attention` section with one card per blocked file and direct CTA buttons.
- [ ] Add an `In progress` section for work that is advancing automatically.
- [ ] Add a compact `Recently completed` section instead of another dense queue table.
- [ ] Make every blocker CTA deep-link to the queue in the right filtered and selected state.

## Deliverables

- New `Today` page layout and copy.
- Readiness strip backed by live app state.
- Blocker cards with direct next-step actions.
- Compact recent-completions and in-progress presentation.

## Dependencies

- Depends on [Phase 1](phase-1-information-architecture-and-shell.md).
- Should align closely with [Phase 2](phase-2-onboarding-redesign.md) so setup and operational readiness use the same model.

## Validation

- [ ] Verify the landing page distinguishes waiting-for-user work from automatic progress.
- [ ] Verify blocker CTA links land in the correct queue context.
- [ ] Verify recent completion and active upload sections remain truthful and scannable.
